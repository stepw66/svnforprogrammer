# บทที่ 6.Branching #

ตอนนี้ท่านจะได้พบกับเทคนิคการบริหารทีมพัฒนาโปรแกรมด้วยการแตก Branch ที่ผ่านมาเมื่อครั้งยังไม่มี SVN เกิดขึ้นบนโลกนั้น การจะเอาทีม 2 ทีมมาทำงานร่วมโปรเจ็กเดียวกัน เป็นการสร้างความยุ่งยากลำบาก เพราะไหนจะต้องตกลงกันเป็นมั่นเป็นเหมาะว่าจะทำงานกันยังไง แบ่งส่วนชัดเจน ไม่สามารถเขียนโค้ดเข้ามาคาบเกี่ยวกันได้  เพราะอะไร ก็เพราะว่ามันจะเป็นการเสียเวลาไง อีกทีมก็ไม่รู้ว่าอีกทีมได้เขียนส่วนนั้นไปแล้ว จึงเขียนขึ้นมาซ้ำ

แม้แต่เมื่อหลังเขียนโค้ดเสร็จแล้ว เอาโค้ดมารวมกัน ปัญหาอีกมากมายยังตามมา มีบั๊กบ้างไรบ้าง  สูบพลังและเรี่ยวแรงของโปรแกรมเมอร์ไปมหาศาล

แต่เมื่อมีระบบ branching ขึ้น ทีมสามารถแชร์โค้ดกันได้ตลอดเวลาหากอยากจะทำ ยิ่งใช้จะยิ่งเห็นประโยชน์  

สิ่งที่จะได้เรียนรู้จากบทนี้ :

- การแตก Branch ทำอย่างไร และมีประโยชน์อย่างไร
- โครงสร้าง Branch สากล
- การรวมโค้ดจาก 2 Branch เข้าด้วยกัน
- การทำ tag point เพื่อระบุตำแหน่ง release version
