# SVN อะไรคือ #

SVN เป็นโปรแกรมตัวหนึ่งที่อยู่ในกลุ่ม SCM หรือ Source Control Management (บริหารจัดการซอร์สโค้ด) มันจะคอยทำหน้าที่เหมือนเครื่องจัดเก็บ Backup แต่ไม่ใช่ Backup อย่างที่เราคุ้นเคย แบบก้อปปี้ทั้งโฟลเดอร์แล้วเปลี่ยนเป็นชื่อใหม่พร้อมระบุวันไว้ ไม่ใช่ Backup ของ SVN จะเหมือน Save Point มากกว่า แบบที่เขียนๆโปรแกรมไป ทดสอบ เห็นว่าโค้ดทำงานได้อย่างถูกต้องแล้ว ก็ทำการ Save Point มันไว้ พร้อมคำอธิบายว่าทำอะไรกับโค้ดลงไป คนจะทำหน้าที่กำหนดว่าจะเก็บ Save Point แล้ว SVN จะทำหน้าที่รวมรวมรายละเอียดว่าแก้ไขโค้ดบรรทัดไหนไปบ้าง แก้อย่างไร เป็นการเพิ่ม หรือแก้ไข หรือลบออก 

เมื่อเขียนโปรแกรมไปนานๆ เราสามารถย้อนกลับมาคลิกดู Save Point จุดต่างๆได้ ไม่เพียงเท่านั้นเราสามารถกำหนดได้ว่า ณ ตำแหน่งตรงไหนคือโปรแกรมเวอร์ชั่น 1.0 ณ ตรงไหนคือโปรแกรมเวอร์ชั่น 2.0 ที่เพิ่มฟีเจอร์ใหม่เข้าไปแล้ว หากลูกค้าไม่ได้ต้องการฟีเจอร์ใหม่ เราก็สามารถ export โค้ดตรงตำแหน่งเวอร์ชั่น 1.0 แล้วส่งขายให้เขาในราคาถูกได้

ที่กล่าวมาข้างต้น เป็นเพียงคุณสมบัติคร่าวๆเท่านั้น SVN ยังมีสิ่งที่มีประโยชน์มหาศาลอีกเพียบ ติดตามได้ในบทต่อๆไป


## ภาพรวม SVN ##

ผมอยากให้ดูภาพนี้ก่อน แล้วเดี๋ยวจะอธิบาย ภาพนี้บอกความเป็น SVN ได้ดีที่สุดแล้ว

![ภาพจาก http://commons.wikimedia.org/wiki/File:SVN_Server_Client_Structure.png](images\chapter1\svn_server_client_structure.png)

โมเดลการทำงานของ  SVN นั้นเป็นโมเดลที่เขาเรียกว่า Client Server คือจะต้องมีองค์ประกอบ 2 ส่วน (ดูภาพประกอบด้วยนะเฮีย)

**Server** จากภาพประกอบจะเห็นว่า มันจะต้องมี SVN เซิฟเวอร์เป็นตัวเก็บ Repository ซึ่ง Repository นี้ก็คือโปรเจ็กหนึ่งๆ ของเรา 

**Client** SVN Client ก็คือเครื่องของโปรแกรมเมอร์ที่จะเชื่อมต่อเข้ามายัง SVN Server เพื่อส่งโค้ดที่เขียนเสร็จแล้วเข้ามาเก็บ หรือ อัปเดตโค้ดที่ SVN Client เครื่องอื่นในทีมส่งเข้าไปเก็บไว้ที่ SVN Server

เอาล่ะผมขอเล่าใหม่อีกที ขอให้ทำความเข้าใจเบื้องต้นนะครับว่า SVN Server กับ SVN Client สามารถเป็นเครื่องๆเดียวกันได้ มันเป็นโปรแกรม 2 ตัวที่จะคุยกัน ตอนแรกเราจะติดตั้งโปรแกรม SVN Server ไว้ที่เครื่องก่อน จากนั้นสร้าง Repository ขึ้นมา เหตุที่เขาไม่เรียกว่า โปรเจ็ก เพราะว่าจริงๆแล้ว ใน Repository นั้นสามารถส่งอะไรเข้าไปเก็บก็ได้ ไม่ใช่เพียงแค่ Source Code เท่านั้น รูปภาพ ไฟล์เวิร์ด เอ็กเซล ได้หมด หลังจากสร้าง Repository แล้วเราก็ติดตั้ง โปรแกรม SVN Client ลงในเครื่อง จากนั้นทำการ Clone โค้ดลงมาจาก Repository ถ้าหากเป็นโปรเจ็กใหม่ๆยังไม่มีโค้ดใดๆ มันก็เปล่าๆ ลงมาในเครื่องของเราตำแหน่งที่เก็บโปรเจ็ก หากเราใช้ xampp มันก็ใน htdocs ทีนี้เมื่อมีอะไรเปลี่ยนแปลงไปในโฟลเดอร์โปรเจ็กของเรา SVN มันจะจับได้แล้ว เมื่อต้องการทำ Save Point เราก็ Commit โค้ดขึ้นไปเก็บไว้ที่ Repository 

ในกรณีทำงานกันเป็นทีมๆนึงหลายคนในโปรเจ็กเดียวกัน ทุกๆคนก็ Clone โปรเจ็กลงมาจาก Repository เดียวกัน ตอนนี้จะมีคน Commit หลายคนแล้ว ตัว SVN Client มันจะคอยตรวจสอบเองว่าหาก Repository มีการเปลี่ยนแปลง มันจะสั่งให้เรา Update Local working คือมันจะพยายามซิงก์ให้โค้ดของโปรแกรมเมอร์ทุกๆเครื่องเหมือนกัน นั่นเอง


## ตัวอย่างการวาง Infrastructure ##
**แบบทำงานคนเดียว**  มีเครื่องๆเดียว ติดตั้งทั้ง SVN Server และ SVN Client ไว้ในเครื่องเดียว 

**แบบทำงานร่วมกันหลายคน** อย่างเช่นการทำงานร่วมกันในบริษัท เราจะติดตั้ง SVN Server ไว้ที่เครื่องกลางเครื่องหนึ่ง ส่วน SVN Client ก็ติดตั้งเครื่องใครเครื่องมัน 

ทั้งสองแบบนี้ถึงโครงสร้างดูจะแตกต่างกัน แต่ในการทำงานนั้นไม่แตกต่างกันเลย
